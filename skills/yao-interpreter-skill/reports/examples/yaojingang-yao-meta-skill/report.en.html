02

一句话定位Positioning

这个 Skill 用于把工作流、提示词、访谈记录、文档或笔记转化为可复用的智能体技能，也支持对既有 Skill 做重构、评估、打包和团队分发准备。Create, refactor, evaluate, and package agent skills from workflows, prompts, transcripts, docs, or notes. Use for skill creation, reusable workflow packaging, skill improvement, evals, and team-ready distribution.

解读重点Interpretation focus

  • 它的核心不是生成一段提示词，而是把可重复的工作方式整理成可路由、可复用、可评估的 Skill 包。Its core job is packaging reusable ways of working into routeable, reusable, and evaluable Skill packages.
  • 它强调先写描述字段并测试触发质量，说明这个 Skill 把“何时使用”看作 Skill 成败的第一入口。It treats the frontmatter description and route quality as the first success condition.
  • 它把轻量脚手架模式和高信任治理模式分开，避免把所有流程都做成重治理工程。It separates lightweight Scaffold work from Governed high-trust packages.

适用场景Use cases

  • 从零创建 Skill：当用户提供重复工作流、提示词、访谈记录、文档或笔记，并希望沉淀成可复用能力时使用。Create reusable agent skills from workflows, prompts, transcripts, docs, or notes.
  • 重构已有 Skill：当 SKILL.md 过长、规则散乱、脚本和参考资料没有分层时，用它把入口、references/、scripts/、reports/ 重新分界。Refactor existing skills into lean entrypoints, references, scripts, reports, and gates.
  • 评估与打包团队 Skill：当 Skill 准备给团队复用时，用它判断应采用脚手架、生产、库级还是治理模式，并补齐接口声明、评估用例、清单文件和门禁。Evaluate and package team-ready skills with the lightest reliable operating mode.
  • 处理高信任交付：当 Skill 涉及文件证据、发布关键流程或治理要求时，用它要求输入文件证据、输出契约、回滚边界、信任报告等可审计标签。Handle governed skill packages with explicit evidence, rollback, and trust boundaries.

工作流Workflow

  • 先判断是否真的需要创建 Skill。一次性任务、没有复用价值或没有稳定输出契约时，直接拒绝 Skill 化。For one-off/no reusable process: Do not create a skill; near-neighbor; require repeated use + reusable output contract.
  • 明确任务、输出、排除项、约束、质量标准和最轻量的适用模式。Capture job, output, exclusions, constraints, standards, and the lightest fit.
  • 按外部标杆、用户材料、本地相邻 Skill 的顺序扫描参考，只暴露不确定性或冲突。Scan references in order: external benchmark, user source, local fit; surface only uncertainty or conflict.
  • 先写描述字段并测试路由质量，再按需要增加 references、scripts、reports 和门禁。Write description early, test route quality, then add only earned folders and gates.
  • 只有在确有价值时，才补充输出风险、产物设计、提示词质量、系统模型和后续演进方向。Add output-risk, artifact-design, prompt-quality, system-model, and next directions only when useful.
