# 报告契约

默认报告语言为简体中文。`report.zh-CN.html` 必须内置右上角 `简体 / EN` 语言切换，打开时默认显示中文；英文独立报告必须从同一份 `analysis.json` 渲染，不能重新评分或手工改写导致数字漂移。

## 输出文件

| 文件 | 必需 | 用途 |
|---|---|---|
| `report.zh-CN.html` | 是 | 中文主报告，内置语言切换，面向学习、使用、评估和准入 |
| `analysis.json` | 是 | 结构化事实、指标、评分和摘要 |
| `findings.json` | 是 | 风险、扣分项、建议和证据链 |
| `qa_report.json` | 是 | 语言、链接、数字一致性和生成状态 |
| `summary.md` | 是 | 轻量摘要，可放到 PR 或文档库 |
| `report.en.html` | 可选 | 独立英文报告 |

## 阅读路径

| 层级 | 目标 | 必含模块 |
|---|---|---|
| 30 秒看懂 | 判断是否值得继续读 | 封面、总分、等级、风险、推荐动作 |
| 5 分钟学会使用 | 判断何时用、怎么用 | 一句话定位、适用场景、谨慎场景、输入输出、示例提示词 |
| 30 分钟深度学习 | 学结构、评分、风险和改法 | 结构总览、渐进披露、工作流、脚本依赖、设计模式、评分明细、证据链、安全审查、路线图 |

## 模块说明写法

- 封面标题区必须和右侧评分卡顶边对齐，不能为了居中导致标题低于评分卡。
- Skill 名称下方的描述必须是 2 到 3 行的真实解读，包含背景、核心问题、亮点或使用提醒，不能只写“基于评分模型生成报告”这类通用说明。
- 封面标签优先使用目标相关标签，例如能力类型、关键亮点、主要风险，而不是只放通用流程标签。
- 模块开头必须直接给结论、事实或关键判断，禁止使用“这一节看”“这一节讲”“你可以先看”这类自我说明。
- 避免只写抽象标签，例如“结构总览”“评分明细”；需要把具体结果、风险或判断标准写出来。
- 说明文字面向第一次看 Skill 报告的人，优先使用短句、具体动作和可判断的标准，不写空泛过渡句。
- 英文切换内容可以保留简洁说明，但中文默认视图必须更完整。

## QA 标准

- 总分等于 9 个维度分数之和。
- 关键发现项必须有证据；证据缺失会降低置信度。
- HTML 内部锚点存在，左侧目录能跳转到各模块。
- 中文报告打开时，可见导航、标题、按钮、分析段落和图表标签为简体中文，不能直接暴露英文 frontmatter 原文。
- 顶部右侧存在 `简体 / EN` 语言切换；点击 EN 后能查看英文定位和场景说明。
- 报告不依赖外部 CDN，离线打开仍有样式和图表。
