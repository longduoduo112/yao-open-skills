# Skill 解读评分规则

总分 100 分，所有扣分都必须能追溯到文件、章节、行号、摘录或缺失证据。若命中红线风险，推荐动作必须降级为“暂缓使用”，即使总分仍然较高。

## 一级维度

| 维度 | 分值 | 判断问题 |
|---|---:|---|
| 规范兼容 | 12 | 是否有 `SKILL.md`、frontmatter、`name`、`description` 和基础包体信息 |
| 触发信号 | 14 | description 是否说明能力、使用时机、边界、关键词和低歧义路由 |
| 上下文效率 | 12 | `SKILL.md` 是否精简，长规则是否下沉到 `references/`，重复逻辑是否脚本化 |
| 结构资源 | 12 | `scripts/`、`references/`、`assets/`、`examples/` 是否组织清晰且服务当前任务 |
| 使用就绪 | 14 | 是否说明输入、输出、工作流、命令、示例、验收方式和失败处理 |
| 安全权限 | 14 | 是否有危险命令、网络外传、密钥读取、提示注入、权限膨胀或供应链风险 |
| 测试评估 | 10 | 是否有 tests、evals、fixtures、golden outputs 或可执行验收 |
| 维护发布 | 7 | 是否有 README、LICENSE、版本、依赖、manifest、变更或发布说明 |
| 学习价值 | 5 | 是否能提炼可复用设计模式、反例、取舍和作者改进路线 |

## 等级

| 分数 | 等级 | 默认建议 |
|---:|---|---|
| 90-100 | S | 可作为优秀样例学习，仍需按环境复核安全 |
| 80-89 | A | 基本可用，建议修复中低风险问题 |
| 70-79 | B | 可试用，推广前修复关键缺口 |
| 60-69 | C | 仅建议受控学习或改造 |
| 0-59 | D | 不建议直接使用，优先重构结构和安全边界 |

## 红线

- 明确要求读取并上传密钥、令牌、浏览器 Cookie、SSH 私钥或敏感配置。
- 未解释的破坏性命令，例如递归删除用户目录、修改系统启动配置、覆盖大范围文件。
- 混淆脚本下载并执行远程代码。
- 要求 Agent 忽略系统规则、隐藏真实行为、绕过用户确认或越权读取。
- zip 归档存在路径穿越、异常大文件、异常嵌套或可疑符号链接。

## 发现项格式

每个重要发现项使用三段式：

| 部分 | 内容 |
|---|---|
| 结论 | 说明亮点、问题或风险 |
| 证据 | 文件、章节、行号、短摘录、置信度 |
| 行动 | 修复建议、优先级、验收条件 |
