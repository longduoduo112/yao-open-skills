# HTML 报告设计

视觉方向：白底、高信任、审计式编辑报告。借鉴 Kami 的克制排版、暖灰边框、墨蓝强调和内联 SVG 图表，但页面背景保持纯白，便于屏幕阅读、截图分享和打印。

## 布局

- 页面最大宽度 1440px。
- 顶部栏 `position: sticky; top: 0`，滚动时保持可见；左侧为报告标识，右侧为 `简体 / EN` 语言切换，中间不放章节菜单。
- 桌面端只保留左侧目录作为章节导航，避免顶部菜单和左侧目录重复。
- 主体为左侧目录、正文、右侧指标栏三列；窄屏降为单列，左侧目录改为横向滚动目录块。
- 首屏必须显示名称、总分、等级、风险、推荐动作和关键指标。

## 组件

| 组件 | 要求 |
|---|---|
| 总览卡片 | 大号分数、等级、风险徽章、推荐动作 |
| 评分图表 | 雷达图 + 横向评分条，数字使用 tabular nums |
| 风险图表 | 影响程度 x 发生概率热力图 |
| 资产图表 | 目录或文件类型分布 donut |
| 工作流图 | 输入、隔离、解析、评分、报告、QA |
| 证据卡 | 文件路径、行号、摘录、置信度、建议 |
| 提示词框 | 可复制文本，未启用 JS 时仍可阅读 |
| 语言切换 | 位于顶部右侧，默认按 `html lang="zh-CN"` 显示中文，点击 EN 后切换英文文本 |
| 附录 | 原始指标、文件树、解析日志，可折叠 |

## 色彩与排版

- 背景：`#ffffff`。
- 主色：`#1B365D`，用于关键分数、导航激活态和少量图表焦点。
- 边框：暖灰 `#e6e0d4`、`#eee9df`。
- 文本：近黑 `#151515`、正文 `#2f2d29`、说明 `#68625a`。
- 成功、警告、危险使用克制语义色，不大面积铺色。
- 卡片圆角不超过 8px，避免嵌套卡片。
- 表格数字右对齐，文本左对齐，长路径允许换行。

## 图表规则

- 图表必须解释结构、场景、风险或评分，不做装饰。
- 优先使用内联 SVG，无外部 CDN。
- 一个图表只强调 1-2 个焦点，避免多色杂乱。
- SVG 文本不能被 viewBox 裁切。

## 打印

- 隐藏复制按钮、语言切换和交互控件。
- 保留标题、分数、证据、建议和来源路径。
- 避免重要卡片跨页断裂。
