# Copyright Comment Policy

## Standard Markdown Notice

Use this for `README.md` and Markdown files without YAML frontmatter:


For `SKILL.md` or any Markdown file with YAML frontmatter, keep the frontmatter at the very beginning of the file and insert the HTML comment immediately after the closing `---` block. This preserves skill routing.

## Target Priority

Add comments in this order:

1. Root `SKILL.md` and `README.md`.
2. Core methodology Markdown: `references/*.md` and `docs/*.md`.
3. Prompt, template, and example Markdown: `prompts/*.md`, `templates/*.md`, and `examples/*.md`.
4. Executable scripts in `scripts/*.py`, `scripts/*.js`, `scripts/*.ts`, and `scripts/*.sh`.
5. Comment-safe config files such as `*.yml`, `*.yaml`, and `*.toml`.

## Skip Rules

Do not add comments to:

- JSON files such as `package.json`, `tsconfig.json`, and `settings.json`
- lock files such as `package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`, `uv.lock`, and `poetry.lock`
- generated files that say `@generated`, `auto-generated`, `generated by`, or `do not edit`
- build outputs, caches, virtual environments, `node_modules/`, `.git/`, and `reports/`

If a file already has a Yao copyright notice near the top, leave it unchanged to avoid duplicate headers.
